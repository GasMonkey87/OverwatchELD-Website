OverWatch ELD Setup Guide

Click a section on the left to open setup instructions.
Setup Sections

1. Download OverWatch ELD

Go to the downloads page and click Download OverWatch ELD.

Open Downloads Page Download Installer

2. Run the Installer

  1. Open your Downloads folder.
  2. Double-click OverWatchELD-win-Setup.exe.
  3. Follow the installer prompts.
  4. Wait for installation to complete.

3. Windows Security Warning

Windows may show a warning because the app is new or not signed yet.
  1. Click More info.
  2. Click Run anyway.
  3. Allow the installer to continue.

4. Open OverWatch ELD

After installation, open OverWatch ELD from:

Start Menu → OverWatch ELD

You may also have a desktop shortcut.

5. Use the Correct Bot Address

When setting up the app, use this Bot API address:

https://overwatcheld.up.railway.app

6. Add the Discord Bot to Your Server

Click the button below, choose your Discord server, and authorize the bot.

Add Discord Bot to Server

7. Link Your Discord / VTC

  1. Open your Discord server.
  2. Type !link in a channel where the bot can respond.
  3. The bot gives you a link code.
  4. Open OverWatch ELD.
  5. Enter the link code when prompted.
  6. After linking, ELD connects to your VTC/Discord server.

8. Companion App Access

The companion app runs locally from the ELD on port 5234.

On the same PC:

http://127.0.0.1:5234

On a phone/tablet using the same Wi-Fi:

http://YOUR-PC-IP:5234

9. Allow Companion Through Firewall

If your phone cannot connect to the companion app, open PowerShell as Administrator and run:

New-NetFirewallRule -DisplayName "OverWatchELD Companion 5234" -Direction Inbound -Protocol TCP -LocalPort 5234 -Action Allow

10. Auto Updates

OverWatch ELD only needs to be installed once.

  1. Open OverWatch ELD.
  2. The app checks for updates automatically.
  3. If a newer version is found, it downloads in the background.
  4. When prompted, click Yes to restart and finish updating.

11. Install & Run ATS Telemetry Server

OverWatch ELD needs the telemetry server running so it can read American Truck Simulator data like speed, location, fuel, damage, and job/load status.

Step 1: Download the Telemetry Server

  1. Open the OverWatch ELD downloads page.
  2. Download the ATS Telemetry Server/Reader if it is listed separately.
  3. If it is included with OverWatch ELD, continue to the next step.

Step 2: Install the ATS Telemetry Plugin

  1. Close American Truck Simulator.
  2. Open the telemetry server folder.
  3. Copy the telemetry plugin files into your ATS plugins folder.
  4. The common ATS plugin folder is:

Documents\American Truck Simulator\plugins

If the plugins folder does not exist, create it.

Step 3: Start the Telemetry Server

  1. Run the telemetry server as Administrator if needed.
  2. Make sure it starts without errors.
  3. Leave it running while using OverWatch ELD and ATS.

Step 4: Start American Truck Simulator

  1. Open ATS.
  2. Load into your profile.
  3. Enter the truck.
  4. Drive or start a job so telemetry data becomes active.

Step 5: Connect OverWatch ELD

  1. Open OverWatch ELD.
  2. Go to the main dashboard.
  3. Verify speed, duty status, location, or telemetry status updates.
  4. If using VTC mode, make sure your bot address is:

https://overwatcheld.up.railway.app

Troubleshooting

  • If ELD shows no telemetry, make sure ATS is open and you are inside a truck.
  • Make sure the telemetry server is running.
  • Restart ATS after installing the plugin.
  • Run the telemetry server as Administrator if Windows blocks it.
  • Check that firewall or antivirus is not blocking the telemetry server.
ATS must be restarted after installing the telemetry plugin. ELD cannot read ATS telemetry until the plugin is loaded by the game.

12. Connect to Vehicle (Start Driving)

OverWatch ELD only receives data when you are inside a truck and actively driving or on a job in American Truck Simulator.

Step 1: Launch ATS

  1. Start American Truck Simulator.
  2. Select your profile.

Step 2: Enter a Truck

  1. Load into the game world.
  2. Make sure you are inside a truck.
  3. If you are in the menu or map screen, telemetry will not send.

Step 3: Start a Job or Drive

  1. Pick up a load from Freight Market or Cargo Market.
  2. Or simply start driving your truck.
  3. Telemetry activates once the truck is moving or engine is on.

Step 4: Verify Connection in ELD

  1. Open OverWatch ELD.
  2. Go to the dashboard.
  3. Confirm you see:
  • Speed updating
  • Duty status changing (Driving / On Duty)
  • Location or map updates
  • Truck name / data

Common Issues

  • ELD shows no data → You are not in a truck yet
  • Speed = 0 → Engine is off or parked
  • No updates → Telemetry server is not running
  • No connection → ATS plugin not installed correctly
OverWatch ELD does NOT connect to menus. You must be inside a truck and actively in-game for telemetry to work.

Pro Tip

The fastest way to confirm everything works is:

  1. Start ATS
  2. Enter a truck
  3. Drive forward
  4. Watch the ELD dashboard update in real time

13. VTC Window Navigation

The VTC window is where drivers and managers can access company tools, live tracking, roster information, maintenance, performance, and Discord-connected features.

Step 1: Open the VTC Window

  1. Open OverWatch ELD.
  2. Log in normally.
  3. From the main dashboard, click VTC.
  4. If you do not see VTC tools, make sure you linked with !link first.

Step 2: VTC Home

The VTC home page shows your company overview, status cards, announcements, and quick access buttons.

  • Announcements: messages from your VTC/Discord server.
  • Fleet Snapshot: quick fleet status overview.
  • Activity Feed: recent VTC activity.

Step 3: Live Map

  1. Click Live Map.
  2. View active drivers and vehicle locations.
  3. Use convoy mode if available to view route/driver tracking.
  4. Use the back button to return to the VTC page.

Step 4: Roster

  1. Click Roster.
  2. View drivers, Discord names, ranks, performance, and achievements.
  3. Admins/managers may see extra actions such as profile tools, awards, or management options.

Step 5: Maintenance

  1. Click Maintenance.
  2. Review truck condition, service status, inspection status, and fleet maintenance needs.
  3. Managers can use this area to help track fleet readiness.

Step 6: Performance

  1. Click Performance.
  2. Review driving stats, miles, loads, safety, and VTC performance tracking.
  3. Use this page to compare driver progress and company activity.

Step 7: Dispatch / Loads

  1. Open dispatch or load tools from the VTC/dashboard area if enabled.
  2. View assigned loads, active loads, pickup status, and delivery tracking.
  3. Some VTCs may post load activity to Discord automatically.

Step 8: Admin / Manager Tools

Some buttons only appear for users with the correct Discord/VTC role.

  • Owner/Admin: can manage settings and VTC tools.
  • Manager: may access roster, fleet, dispatch, or maintenance controls.
  • Driver: can view driver tools and assigned VTC features.
If a VTC button is missing, your account may not be linked, your Discord role may not have access, or the VTC server may not be configured yet.

Troubleshooting

  • If VTC says Standalone, run !link again and enter the new code.
  • If Live Map shows no drivers, make sure telemetry is running and drivers are inside ATS trucks.
  • If announcements do not appear, verify the Discord bot is online and configured for your server.
  • If admin tools are missing, check your Discord role permissions.

14. Shortcut Keys

OverWatch ELD includes keyboard shortcuts to quickly control the app without using the mouse.

  • Ctrl + Shift + R → Restart ELD

Navigation

  • Ctrl + 2 → Logs

VTC Quick Access

Duty Status (Quick Change)

Companion / Tools

  • Ctrl + Shift + T → Restart Telemetry
Some shortcuts may depend on your version or user role. Admin features may not be available to all users.

Pro Tip

Use shortcuts while driving to quickly switch duty status or open tools without leaving your workflow.

15. Check Open Ports (Companion Verification)

This step verifies that the OverWatch ELD Companion App is running and accessible on port 5234.

Step 1: Check Port Using PowerShell

Open PowerShell as Administrator and run:

Get-NetTCPConnection -LocalPort 5234

If the companion is running correctly, you should see:

State: Listen

Step 2: Verify in Command Prompt

Alternatively, open Command Prompt and run:

netstat -ano | findstr :5234

Expected output:

TCP 0.0.0.0:5234 LISTENING

Step 3: Identify the Program (Optional)

If needed, find what program is using the port:

tasklist | findstr [PID]

Replace [PID] with the number shown from the netstat command.

Step 4: Test in Browser

On your PC, open:

http://127.0.0.1:5234

On your phone (same Wi-Fi):

http://YOUR-IP:5234

Important Notes

  • 0.0.0.0:5234 → Works for phones and other devices
  • 127.0.0.1:5234 → Works only on your PC
  • If nothing shows → Companion app is not running
If the port is not open, restart OverWatch ELD and make sure the Companion API started successfully.

16. Discord Bot Commands

Use these commands inside your Discord server where the OverWatch ELD bot is installed.

Getting Started

  • !link → Generate a code to connect your ELD to the VTC
  • !unlink → Disconnect your ELD from the VTC
  • !status → Check bot status and connection

Driver Commands

  • !myinfo → View your driver stats and info
  • !logs → View your recent logs/activity
  • !loads → View your active or recent loads

Dispatch Commands

  • !dispatch → Open dispatch menu or send message
  • !reply → Reply to dispatch messages

Admin / Setup Commands

  • !setdispatchchannel → Set dispatch channel
  • !setbolchannel → Set BOL channel
  • !setlogschannel → Set logs channel
  • !setinspectionchannel → Set inspection channel
  • !setleaderboardchannel → Set leaderboard channel

System Commands

  • !help → Show available commands
  • !ping → Check bot response time
Some commands require Admin, Owner, or Manager roles in your Discord server.

Tips

  • Use commands in a channel where the bot has permission to read and send messages.
  • If a command doesn’t work, check bot permissions and roles.
  • Make sure your ELD is linked using !link first.

Need Help?

If setup does not work, verify:

  • The Discord bot was added to your server.
  • You used the correct bot address.
  • You entered the correct !link code.
  • Your firewall allows port 5234 for companion access.
